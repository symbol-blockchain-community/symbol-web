## 摘要

2025年9月23日，官方Symbol區塊鏈X帳號 （[@thesymbolchain](https://x.com/thesymbolchain)）宣布進行重大[網路升級](https://x.com/thesymbolchain/status/1970323150253781038)。

此升級預計將於區塊高度 `4,759,100` 時進行，預估時間為 9 月 25 日世界協調時間晚上 11:30。

此升級旨在解決Symbol網絡內潛在的攻擊途徑，並鼓勵節點運營商盡快實施更新。

## 升級詳情
本次網路升級透過修復特定條件下可能導致聚合交易雜湊值失去唯一性的漏洞，解決了潛在攻擊途徑。
同時整合了更新的依賴庫（CPPZMQ 4.11.0、RocksDB 10.5.1 及 OpenSSL 3.5.2），從而提升網路整體穩定性與安全性。
此外，Rest Gateway 版本 2.5.1 新增對聚合交易 v3 的支援，並解決 WebSocket 交易中 maxFee 屬性缺失的問題。
由於這些更新屬於重大分叉（BREAKING FORK），節點運營者必須事先進行準備工作。

- [**Catapult Client v.1.0.3.9**](https://github.com/symbol/symbol/releases/tag/client%2Fcatapult%2Fv1.0.3.9)
  - Fixed:(BREAKING FORK) fix aggregate transaction hash bug where the aggregate transaction hashes were not unique in certain circumstances.
- [**Rest Gateway v2.5.1**](https://github.com/symbol/symbol/releases/tag/rest%2Fv2.5.1)
  - Fixed:WebSocket transaction is missing the maxFee property, which was getting dropped.


## 開發人員資訊
建議更新 JavaScript 與 Python SDK 至 3.3.0 版本、舊版 TypeScript/JavaScript SDK 至 2.0.6 版本，以及 Java SDK 至 1.0.4 版本。此更新旨在確保與去中心化應用程式（dApps）的相容性。建議使用者於目標區塊高度達成後發布其應用程式。

此外，更新版本的節點設定工具 Symbol-Bootstrap（版本 1.1.12）與 Symbol-Shoestring（版本 0.2.2）已於 GitHub 發布，以利快速升級。

## 影響與對策
此升級顯著提升了Symbol網絡的安全性，並解決了潛在的攻擊途徑。
該漏洞在特定條件下可能導致聚合交易雜湊值非唯一，惡意攻擊者可能藉此破壞網絡運作。修正此問題後，網絡可靠性獲得提升。

平台持續的穩定性將有助於企業用戶的業務延續性。然而，忽視更新的節點運營商與開發者，則面臨被網絡分叉淘汰的風險。

由於升級時間取決於區塊高度，實際執行時機可能因網路進度而有所差異。建議您透過Symbol區塊鏈官方X帳號（[@thesymbolchain](https://x.com/thesymbolchain)）或[Symbol社群網站](https://symbol-community.com)獲取最新資訊。
