## 1. 關於 `SIP`

`SIP` 是 "Symbol Improvement Proposal" 的首字母縮寫，是改進 Symbol 區塊鏈規格或功能的提案，是其應用形式，即 Bitcoin 的 `BIP` 和 Ethereum 的 `EIP`。

目前，關於改進 Symbol 區塊鏈規格和功能的建議可以以任何形式向覈心團隊提出，例如在 Discord上提出，沒有正式的格式，如 "BIP"
或 "EIP"。

因此，本 `SIP-1000` 也不採用正式格式，而是繼 `BIP` 和 `EIP` 之後，作為 `SIP-1000` 提出，並就其格式征求意見。同時，作為 `SIP` 提出提案流程編寫指南，並就其格式征求意見。

請註意，`SIP-1000` 仍處於公眾反饋階段，不會根據第 X 次會議上提出的意見或其匯總情況來決定。

會議還宣佈，經討論後，計劃與覈心小組一起編寫 `SIP-1` 草案，其中包括實際正式審議和決策的流程和規則。

而且，如果 Symbol 區塊鏈也能為提案提供一種 `SIP` 格式，如 `BIP` 或 `EIP`，作為一套準則，用戶今後想提出提案時就可以使用，也更容易看到每項提案的討論和結論。

#### 補充信息
`SIP-1000` 中的 `1000` 看起來像一個提案編號，但並不意味着第 `1000` 個提案，其含義也不是很深，因為還沒有正式的提案流程。提案者避免使用兩位或三位數字，因為這些數字通常與計算機科學中的保留或知名主題有關。


## 2. `SIP-1000`：關於將 "由聚合交易組成的數據的metadata"標準化的建議

在 Symbol 區塊鏈上創建 fully-on-chain NFT 時，通常會使用 Aggregate Transactions 來分割和記錄數據，或者在記錄無法存儲在單個 Transaction 中的數據時使用其他方法。

不過，有關數據格式和分割方式的信息也取決於數據的創建和解碼方式。

因此，建議將 metadata 標準化，以便在記錄 NFT 數據鏈時，更好地將用於解碼 NFT 圖像的信息和其他數據存儲在一起，並使數據存儲格式易於用戶理解。

### 為什麼要提出 metadata 標準化？

目前，Symbol 已擁有許多 fully on-chain 的數據記錄方法，如 NFTDrive、metal 和 COMSA，以及用於引用這些數據的解碼方法。

每種方法都有自己的特點和優點，但事實上，有許多不同的方法也會帶來各自的挑戰。 

要檢索存儲在 NFT 中的圖像等數據，需要根據數據存儲格式進行解碼處理。不過，如果用戶確切知道他們試圖訪問的 fully-on-chain NFT 是以什麼格式存儲的，那麼創建解碼器就會更容易。

這就是提出 metadata 標準化的原因。


### 受益於更簡便的解碼器創建

對於處理 NFT 的產品和服務而言，更容易創建解碼器可帶來以下好處

* NFT 數據的明確解碼方法一旦標準化，用戶將立即知道目標內容的格式，從而更容易選擇適當的解碼器。這將促進對 NFT 內容的使用。
* 標準化有望減輕創建解碼器的負擔，使開發人員能夠更快、更有效地專註於產品開發和服務交付，並改善 UI/UX，使用戶能夠輕鬆訪問 NFT 內容。
* 預計這將通過提高便利性來振興符號生態係統。

### 關於 `SIP-1000` 的說明
如果 `SIP-1000` 存儲在與實際 NFT 數據不同的位置，NFT 創建者必須設計並明確指定 `SIP-1000` 的位置，以便錢包等解碼應用程序能夠讀取。

一般情況下，fully on-chain NFT 數據分別存儲在標頭信息和實際 NFT 數據中。標頭信息包括實際 NFT 數據內容、大小和圖像文件描述。

這些描述對於正確解碼實際 NFT 數據至關重要，`SIP-1000` 的作用與完全鏈上 NFT 的標頭信息相同。通過預加載 `SIP-1000` 數據，可以準確分析實際 NFT 數據結構並確定適當的解碼器。

另一方面，由於有多種存儲、記錄和配置 NFT 數據的方法，`SIP-1000` 並未規定應在何處寫入這些信息，而是建議將數據結構標準化。數據結構的標準化應另行討論。
