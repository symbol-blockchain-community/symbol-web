# March 2026 Blog (Japanese Translation)

---

Hi everyone! It has been yet another intense month of working nonstop to finish the documentation... but we finally did it. The new Symbol documentation will be released in just a few weeks!

The number of tutorials produced has once again set a new record. The progress graph has become so extreme that it is almost funny.

For two months I have poured all my energy into documentation work, but I am finally ready to return to XYM City. Honestly, my brain feels like it might explode 😄

Notes are scattered everywhere: Google Drive, Notion, text files, my phone, and even Post-its beside my pillow.

But before that, here is one last little report. It is only graphs, with no screenshots. Hopefully this really will be the last one.😁

## Summary of Work Done

- **Content Writing**
**Developer Page**(task-docs-dev)：:task-docs-dev: We created tutorials about [mosaic restrictions](https://docs.symboltest.net/en/devbook/mosaics/mosaic-restrictions/) and [network time](https://docs.symboltest.net/en/devbook/chain/network-time/).

  **User Page**(task-docs-user)：We wrote several small pages, including the [security page](https://docs.symboltest.net/en/userbook/security/) linked from the wallet. We also started creating a Shoestring guide for deploying and operating a node.
  
- **Review**
**Developer Page**(task-docs-dev)
[Zero](https://github.com/zero4862) published new tutorials on proving the definitions of on-chain [transactions](https://docs.symboltest.net/en/devbook/chain/prove-transaction/) and [mosaics](https://docs.symboltest.net/en/devbook/chain/prove-mosaic-definition/) reported by nodes, and also provided multiple tutorials on [WebSockets programming](https://docs.symboltest.net/en/devbook/websockets/listen-new-blocks/).

  All of them were reviewed and then published.
  
  **NEM**
  After fully rewriting the specification for [NIS1 REST OpenAPI](https://github.com/NemProject/nem/tree/dev/openapi) last month, [CryptoBeliever](https://github.com/cryptoBeliever) started working on the specification for [SymbolOpenAPI](https://github.com/symbol/symbol-openapi/). This time it covers 430 files and 6,650 lines of YAML. After four weeks of review exchanges, it is now almost complete!

- **Management**

## Documentation Progress Status

This is the current status of all documentation tasks.
Please pay special attention to the top item, "Symbol Documentation." The other items are listed as its subtasks. The bars in the chart represent the number of completed subtasks. Those completed this month are shown in **orange**.

<img width="669" height="377" alt="Image" src="../images/Mar2026_DocsBlogs_2.webp" />

We did it! All tutorial categories are now at 100%!
In fact, the Analytics and Bridge tutorials were moved to the post-release phase, so the only remaining tutorials are the cross-chain trading tutorials. And those are already sitting on my desk, waiting for review.

For the reference guide, it is enough to redirect links from the old documentation to the new pages.

The tasks still remaining in the Infrastructure category are all related to publishing the new documentation and making sure features such as continuous integration (CI) continue to work under the new URLs.

And finally the user manual is currently about 30% complete. The delay is mainly due to the ongoing Shoestring guide, and because development of the new wallet and explorer is not far enough along yet.

So! 85% of the Symbol documentation we planned is already complete, which means these documents will soon become official. And the next progress report should look quite different 😁

## What’s Next

Once I clear the large pile of items waiting for review, I plan to publish the documentation.

After that Zero will update the old NEM documentation, and I will focus on reviewing those. That should also give me plenty of time to work on that thing.

Of course, as soon as the new wallet and explorer are complete, I will create guides for them too.

Thank you very much for reading this far. See you again next month!
