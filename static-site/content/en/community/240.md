## Compatibility with multiple blockchains, including Ethereum (EVM)

[NFTDrive Inc.,](https://nftdrive.net/) (based in Kyoto, Japan; CEO: Michio Nakashima ) has announced , a Japanese NFT-specific wallet [NFTDriveEX](https://nftdrive.net/?nav=ex) which is currently being released, will support multiple chains, including Ethereum (EVM) and Layer 2.
![NFTDriveEX_2.png](../images/NFT_Drive_EX_2_a2882a24b6.png)

NFTDriveEX has a simple, easy-to-use interface and is widely used.

This new feature is designed to make NFTDriveEX even easier to use.

Multi-chain is a technology that allows crypto assets and NFTs, such as Bitcoin, to interact and transfer between different blockchains, thereby compensating for potential blockchain issues such as cost, scale and volume of distribution.

It is also said that a lot of NFT data is currently off-chain and at risk of being lost. (※１)

Symbol blockchain, which does not require an EVM (Ethereum Virtual Machine ※2) to use the blockchain, eliminating the unexpected behaviour of smart contract. In Symbol, NFTs are easily minted as fully on-chain NFTs, is stored as on-chain data, the system is said to take the reliability of traditional NFTs to another level by storing them stably as data and making them.

Currently, NFTDrive offers a 'hybrid fully on-chain' technology that uses the fully on-chain data stored on Symbol as master data and as a reference for other chain NFTs that can be listed on the major NFT market press.
It also allows NFTs minted and owned by chains other than Symbol to be easily switched to Symbol, enabling freer and more open P2P trustless trading inherent to blockchain without having to go through third-party marketplaces.

In terms of payment, by supporting ETH, many ERC20 tokens on Ethereum (layer 2) and stable coins, it will be possible to mint NFTDrive fully on-chain NFTs flexibly on layers that do not have Symbol (XYM) and the traditional Symbol chain, and similarly, functions such as viewing and sending NFTs on Ethereum and other chains can be used without modification.


(※１)This is because many NFTs technically only store on the blockchain the reference URL of the server that stores in 'off-chain' digital data, such as images, associated with the NFT.

(※２)The main body of the program that executes smart contract on the blockchain. It only executes the source code, so it is not determined whether the result is correct for the service. It is therefore necessary to audit the behaviour of the source code when using it.
