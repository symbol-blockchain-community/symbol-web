# 2026年2月のブログ（日本語訳）

あの… ちょっと… 一息つかせてください… あと少しだけ… 😮‍💨 

はい皆さんこんにちは! 2月は本当にハードでした。 書いてはレビューの繰り返しで休みなく働いて、開発者向けチュートリアルの本数も過去最多を叩き出しましたよ！

ですが、まだ終わりじゃありません。つまり、私が…そう、あの大好きな街に戻れるには、まだもう少しこの調子が続きそうです。

結果として、今回のレポートはかなり地味です。プロットだけでスクリーンショットはありません。わかりますブーイングですね。

## 行った作業の概要

- **コンテンツ執筆**
**開発者ページ**(task-docs-dev)：マルチシグアカウントの[作成](https://docs.symboltest.net/en/devbook/accounts/configure-multisig/)と[署名](https://docs.symboltest.net/en/devbook/accounts/sign-multisig/)、[トランザクションのスポンサー（ガス代肩代わり）](https://docs.symboltest.net/en/devbook/transactions/fee-sponsorship/)、[アカウント制限の追加](https://docs.symboltest.net/en/devbook/accounts/account-restrictions/)に関するチュートリアルを作成しました。

  **開発者ページ**(task-docs-dev)：リファレンスガイドに[シリアライゼーション（直列化）の表](https://docs.symboltest.net/en/devbook/reference/serialization/)と[WebSocket API](https://docs.symboltest.net/en/devbook/reference/websockets/)を追加しました。
  
  **テキストブックページ**(task-docs-textbook)：テキストブックに[メタデータ](https://docs.symboltest.net/en/textbook/metadata/)のページを追加しました。
- **レビュー**
**開発者ページ**(task-docs-dev)：
[Zero](https://github.com/zero4862) は、ネームスペースを[アドレス](https://docs.symboltest.net/en/devbook/namespaces/link-namespace-to-address/)や[モザイク](https://docs.symboltest.net/en/devbook/namespaces/link-namespace-to-mosaic/)にリンクする方法、[アカウント](https://docs.symboltest.net/en/devbook/namespaces/namespace-metadata/)、[ネームスペース](https://docs.symboltest.net/en/devbook/namespaces/namespace-metadata/)、[モザイク](https://docs.symboltest.net/en/devbook/mosaics/mosaic-metadata/)へのメタデータ追加、[モザイクの新規作成](https://docs.symboltest.net/en/devbook/mosaics/create-mosaic/)、[モザイク定義の更新](https://docs.symboltest.net/en/devbook/mosaics/modify-mosaic-definition/)、[ネームスペースのレシート](https://docs.symboltest.net/en/devbook/namespaces/resolve-namespace-from-receipt/)、[モザイクの回収](https://docs.symboltest.net/en/devbook/mosaics/revoke-mosaic/)、そして[チェーンステート](https://docs.symboltest.net/en/devbook/chain/chain-heights/)に関する新しいチュートリアルを作成しました。いずれもレビューを経て公開されました。

  **NEM**：[CryptoBeliever](https://github.com/cryptoBeliever) は、[NIS1 の REST API](https://github.com/NemProject/nem/tree/dev/openapi) 用に [OpenAPI](https://www.openapis.org/) の仕様を丸ごと書き上げるという素晴らしい仕事をしてくれました。ファイルは180個、YAMLは4,500行超にのぼり、レビューには数週間にわたるやり取りを必要としました。これは、今後公開される新しい NEM ドキュメントで使われる予定です。

  あとは彼に Symbol の REST API でも同じことをやってもらうよう説得するだけです。[今のドキュメント](https://docs.symboltest.net/en/devbook/reference/rest/symbol/)は、まだだいぶ良くできると思うので😊
- **マネジメント**
新しいNEMドキュメントサイトの開発に向けたロードマップを作成し、アートチームからアートディレクションを受けました。

  1か月間、2人のライターがチュートリアル作成にフルタイムで取り組むため、タスクリストやジョブのバッチ化の準備をする時間が必要でした。

## ドキュメンテーションの進捗状況

これは、すべてのドキュメントタスクの現在の状況です。
特にご注目いただきたいのは、一番上の「Symbol Documentation」です。他の項目は、そのサブタスクとしてリストされています。チャートの棒グラフは、完了したサブタスクの数を表しています。今月完了したものが**オレンジ色**で表示されています。

<img width="669" height="377" alt="Image" src="../images/Feb2026_DocsBlog_3_4f517feb.webp" />

ご覧のとおり、さまざまな面で大きく前進しています！ 全体の Symbol Documentation のバーが動いていないように見えるのは、私がそこにタスクを追加し続けていて、その分だけ進捗が一時的に後退しているからです。これらのタスクが完了すれば、上部のバーは一気に大きく進むはずです。😁

結果として2月はこれだけでした。仕事、仕事、仕事で、楽しいことは一切無し。つまり、XYM Cityも無しということです。ですがこれは、実はゲームにとっては良いことでした！少し距離を置けて、新鮮な気持ちで見直すことができました。XYM Cityに戻るときは、爆発的なエネルギーと、そこで語るべきことをもう少し携えて戻ってまいります。

## 次の予定

Zeroと私は、チュートリアルが完成するまでSymbolドキュメントのスプリントを続けます。

その後、彼は新しいNEMのドキュメントに移り、私はレビューに戻って…それからXYM Cityに取り組みます。

また、新しいSymbolウォレットも順調に進んでおり、完成し次第ガイドを書きます。これらは開発者向けチュートリアルよりも簡単です。

まだ抜け出せてはいませんが、トンネルの先に光が見えてきました！しかもそれはネオンカラーの光です。😎

ここまでお読みいただきありがとうございました。また来月お会いしましょう！
