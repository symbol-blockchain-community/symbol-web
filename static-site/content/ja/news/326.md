## 2025年9月のブログ（日本語訳）

今月は忙しかったです。[ネットワークアップグレード](https://x.com/thesymbolchain/status/1970323150253781038)があり、私たちみんなずっと気を抜けずでした！チームはSDKやツール、アプリケーションの更新を急ぎつつ、フォークに関してコミュニティやノード運用者とコミュニケーションを取り対応しました。そしてついに…ネットワークアップグレードは無事に完了しました！🎉

素晴らしいチームです 👏🏽

## 主な業務の概要

- [Desktop Wallet 1.2.0](https://github.com/symbol/desktop-wallet/releases/tag/v1.2.0) と [Explorer v1.4.0](https://github.com/symbol/explorer/releases/tag/v1.4.0)のリリース🎉
- デスクトップウォレットとエクスプローラーのNodewatchへの移行完了。
- SIPワークフローの再構築。
- NEMエクスプローラーバックエンドサービスの継続的な進捗。

## 各業務について

**Symbolデスクトップウォレット**

- https://x.com/NA5GGYbQ9Nm0iLm/status/1952914223933342044
ちょっとしたことかもしれませんが、ご意見しっかり受け止めました 😛。
- デスクトップに新しいウォレットアイコンが表示されるようになりました！

**Nodewatch API**

- Statistic Servicesの全機能がNodewatchへ完全に移行されました。
-  [メインネット](https://nodewatch.symbol.tools/api) 及び [テストネット](https://nodewatch.symbol.tools/testnet/api)へのデプロイメントが完了し、運用を開始しています。
- 利用可能なエンドポイントについては、[Open API ドキュメント](https://nodewatch.symbol.tools/api/openapi) を参照してください。

**スタティスティクスサービスのNodewatch移行**
- 最新リリースでは、デスクトップウォレットとエクスプローラーのどちらもが Nodewatchに完全に統合されました。
- 旧 statistic service をお使いの場合は、nodewatch サービスへの移行をおすすめします。詳しくは [Open API ドキュメント](https://nodewatch.symbol.tools/api/openapi)をご覧ください。


**SIP**
![Anthony_Sep_2025_1.png](../images/Anthony_Sep_2025_1_d2793716c6.png)
- [**旧設計**](https://symbol-community.com/ja/news/278) と比べて、提案者にとって使いやすくなるように、SIP のワークフローを見直しました。
- 主な違いは、提案者がメンテナーから番号の割り当てを待たずに、SIP のプルリクエストを直接作成できるようになった点です。
- PRはまだレビュー待ちですが、アイデアがあったり改善提案をしたい場合は、こちらの [SIP-1](https://github.com/symbol/symbol/blob/sips/SIPs/sip-1.md) をご確認ください。

**NEMエクスプローラーバックエンドサービス**
- コードベース見直し、リファクタリング、ユニットテストを追加。
- ローカル開発により良いツール、DBeaver と OrbStack を見つけました。
- 一部の NIS ローカルエンドポイントは、[ホワイトリストに登録された IP ](https://github.com/NemProject/nem/blob/daee186381374e6721634e14ab15ba7834026584/nis/src/main/resources/config-default.properties#L106C1-L106C23)からしかアクセスできません。これまでは、 `nis.additionalLocalIps` を更新するたびにノードの再起動と再同期が必要で、時間がかかっていました。
- SSHトンネルを使うというより良い解決策を見つけました。NISノードにアクセスできれば、ローカルマシンからリクエストをトンネルして、再起動や再同期をせずに済みます。これならばシンプルで効率的です！
